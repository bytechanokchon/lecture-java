# ประวัติศาสตร์ภาษา Java (Histoty Of Java)

โดยจุดเริ่มต้นของ java เกิดขึ้นมาจาก 2 เหตุผลหลัก คือ

1. เพื่อปรับตัวให้เข้ากับสภาพแวดล้อมและการใช้งานที่เปลี่ยนแปรงไป
2. เพื่อปรับปรุงและพัฒนาศิลปะของการเขียนโปรแกรม

โดยการพัฒนาภาษานี้ จะได้รับแรงผลักดันจาก 2 เหตุผลนี้เป็นหลัก

## วงศ์ตระกูลของภาษา Java (Java’s Lineage)
ภาษา Java มีความเกี่ยวข้องกับ C++ ซึ่งถูกสืบทอดมาจาก C โดยตรง ทำให้คุณสมบัติส่วนใหญ่ของ Java จะถูกสืบทอดมาจากภาษาทั้ง 2 นี้ โดยตัวภาษาได้รับคุณสมบัติเกี่ยวกับไวยากรณ์มาจากภาษา C และคุณสมบัติเกี่ยวกับการเขียนโปรแกรมเชิงวัตถุมาจาก C++

โดยตลอดระยะเวลาที่ผ่านมา ตัวภาษาถูกปรับปรุงมาเรื่อย ๆ (จนปัจจุบันเวอร์ชัน 21) เพื่อแก้ไขปัญหาพื้นฐานของภาษาโปรแกรมรุ่นก่อน ๆ 

## จุดกำเนิดการเขียนโปรแกรมสมัยใหม่: ภาษา C (The Birth of Modern Programming: C)
เมื่อภาษา C เกิดขึ้น มันเปลี่ยนแปรงโลกของการเขียนโปรแกรมไปตลอดการ ทั้งวิธีคิด ประสิทธิภาพ โครงสร้าง ที่่สามารถมาแทนที่ assembly ได้เลยในการสร้างโปรแกรม

โดยเมื่อมีการออกแบบภาษาคอมพิวเตอร์ใหม่ ๆ มา มักจะคำถึงถึงสิ่งต่าง ๆ ดังต่อไปนี้
1. ความง่ายในการใช้งาน เทียบกับพลังงาน (ease-of-use versus power)
2. ความปลอดภัยเทียบกับประสิทธิภาพ (safety versus efficiency)
3. ความแข็งแกร่งเทียบกับความยิดหยุ่น (rigidity versus extensibility)

ก่อนที่จะมีภาษา C โปรแกรมเมอร์มักจะต้องเลือกว่าจะเอาภาษาที่มีคุณลักษณะชุดใดชุดหนึ่งเป็นพิเศษ เช่น
- Fortran ใช้สำหรับเขียนโปรแกรมที่มีประสิทธิภาพพอสมควรได้สำหรับแอปพลิเคชันทางวิทยาศาสตร์ แต่ก็ไม่เหมาะกับการนำมาเขียนระบบขนาดใหญ่
- Basic เรียนรู้ได้ง่าย แต่ไม่ทรงพลัง ขาดโครงสร้างที่่ทำให้การพัฒนาโปรแกรมขนาดใหญ่สามารถทำได้ยาก
- Assembly สามารถใช้งานพัฒนาโปรแกรมประสิทธิภาพสูงได้ แต่ไม่ง่ายที่จะเรียนรู้และใช้งาน และดีบักโค้ดได้ยาก

และภาษาในยุคก่อน ไม่ได้ถูกออกแบบมาสำหรับการเขียนโปรแกรมเชิงวัตถุ แต่จะใช้หลักการ GOTO แทน ทำให้มักจะเกิดปัญหา "โค้ดสปาเก็ตตี้" ซึ่งเป็นกลุ่มของคำสั่งที่กระโดดไปมา และมีเงื่อนไขพันกันยุ่งเหยิง

## ขั้นต่อมาของการเขียนโปรแกรม: C++ (C++: The Next Step)
ตลอดเวลาที่ผ่านมา ภาษา C ได้รับความนิยม และถูกใช้อย่างแพร่หลายจนถึงปัจจุบัน แต่ทำไมต้องมีภาษาโปรแกรมอื่นเกิดขึ้นมาละ ?

ปัญหาของภาษา C คือ มีความซับซ้อน ยิ่งโปรแกรมใหญ่ ก็ยิ่งมีความซับซ้อน เกินความสามารถที่โปรแกรมเมอร์จะรับไหว

เพื่อแก้ปัญหาดังกล่าว จึงได้เกิดภาษา C++ ขึ้น และได้มีการผลักดันโครงสร้างใหม่ขึ้นมา คือ การเขียนโปรแกรมเชิงวัตถุ (Object-Oriented Programming)

## เวทีพร้อมแล้วสำหรับภาษา Java! (The Stage Is Set for Java)
ในช่วงปลายปี 1980 จนถึงต้นปี 1990 การเขียนโปรแกรมด้วยภาษา C++ ได้รับความนิยมเป็นอย่างมาก
ด้วยความสามารถของการเขียนโปรแกรมเชิงวัตถุ ที่ทำให้สามารถสร้างโปรแกรมได้ทุกรูปแบบ 

แต่เมื่อเวลาผ่านไป เทคโนโลยีใหม่ก็เข้ามา โดยเทคโนโลยีใหม่ที่เข้ามาคือ world wide web 
ซึ่งปฏิวัติอุสาหกรรมของการเขียนโปรแกรมคอมพิวเตอร์ไปอีกขั้น

## การสร้างภาษา Java (The Creation of Java)
ภาษา Java ถูกสร้างขึ้นโดย James Gosling, Patrick Naughton, Chris Warth, Ed Frank, Mike Sheridan ที่บริษัท Sun Microsystem ในปี่ 1991

โดยแรงผลักดันหลักที่ทำให้ภาษา Java ถูกสร้างขึ้นมา เกิดจากความต้องการภาษาที่ไม่ขึ้นตรงต่อระบบปฏิบัติการ (Windows, Linux, MacOS) หรือสถาปัตยกรรม สามารถนำโปรแกรมไว้ติดตั้งไว้บนอุปกรณ์อิเล็กทรอนิกส์ต่าง ๆ ได้ เช่น ไมโครเวฟ รีโมท เป็นต้น

โดยในตอนนั้น ภาษาอย่าง C และ C++ พวกมันจะสามารถทำงานได้บนระบบปฏิบัติการอันใดอันหนึ่งเท่านั้น
ถ้าอยากนำไปทำงานบนระบบอื่น จะต้องทำการคอมไพล์ใหม่ สำหรับระบบนั้น ๆ

ในช่วงวเลาที่กำลังมีการพัฒนาภาษา Java สิ่งที่เป็นแรงผลักดันอีกอย่างหนึ่งคือ การเกิดขึ้นของ world wide web โดยหากไม่มีสิ่งนี้เกิดขึ้น ภาษา Java อาจไม่ได้รับความนิยมในช่วงนั้น โดยภาษา Java ทำให้การเข้าถึง world wide web สามารถเข้าถึงได้จากทุกอุปกรณ์

สาเหตุที่ภาษา Java ได้รับความนิยมขึ้นมานั้นคือ
1. ถูกออกแบบ ทดสอบ และหปรับปรุงโดยโปรแกรมเมอร์ที่ใช้งานจริง
2. มีความสอดคล้องและการทำงานที่เป็นไปตามตรรกะ (ทำงานจากซ้ายไปขวา บนลงล่าง)
3. โปรแกรมเมอร์สามารถควบคุมได้เต็มที่ หากเราเขียนโปรแกรมได้ดี โปรแกรมก็จะสามารถทำงานออกมาได้ดี แต่หากเขียนไม่ดีโปรแกรมก็จะทำงานออกมาไม่ดีเช่นกัน

## ก่อกำเนิดภาษาใหม่จากแนวคิดของ Java (The C# Connection)
แนวคิดที่เกิดขึ้นจากภาษา Java กลายเป็นลากฐานการออกแบบสำหรับภาษาใหม่ ๆ ที่กำลังจะเกิดขึ้นบนโลกใบนี้ เพราะความสำเร็จของภาษานี้ ไม่สา่มารถมองข้ามไปได้

ตัวอย่างเช่น การเกิดขึ้นของภาษา C# ซึ่งถูกพัฒนโดวย Microsoft เพื่อรองรับการเกิดขึ้นของ .NET Framework โดยทั้ง C# และ Java มีหลายอย่างที่คล้ายกันมาก ไม่ว่าจะเป็น ไวยกรณ์ การเขียนโปรแกรมเชิงวัตถุ การใช้งานวัตถุที่เหมือนกัน

## การใช้งานที่ยืนยาว (Longevity)
ภาษาใหม่ ๆ ที่เกิดขึ้น ล้วนได้รับแนวคิดจากภาษา Java เช่น Kotlin Ruby Go Scala Rust และภาษาอื่น ๆ แต่ภาษา Java ก็ได้รับการพิสูจน์มากว่าสามทศวรรษนับตั้งแต่ Java เกิดขึ้น คือมันยังคงถูกใช้งานทั่วโลก

## Java ส่งผลกระทบอะไรกับอินเตอร์เน็ต (How Java Impacted the Internet)
Java ช่วยทำให้การเขียนโปรแกรมเกี่ยวกับอินเตอร์เน็ตสามารถทำได้ง่ายขึ้น และยังสะดวกในการพกพาและมีความปลอดภัย

## Java Applets
ในช่วงเวลาที่เกิดขึ้น สิ่งที่น่าตื่นเตร้นที่สุดคือ Java Applets
โดยมันสามารถส่งข้อมูลผ่านอินเตอร์เน็ตและทำงานอัตโนมัติในเบราว์เซอร์ที่รองรับ Java และ Server สามารถส่งฟังก์ชันมาให้ Client สามารถใช้งานได้

## ความมหัศจรรย์ของ Java: ไบต์โค้ด (Java’s Magic: The Bytecode)
Java Byte Code ทำให้เราสามารถรันโปรแกรมที่ถูกเขียนด้วยภาษา Java จากที่ไหนก็ได้ โดยใช้ความสามารถของ Java Virtual Machine ซึ่งเป็นส่วนหนึ่งของ Java Runtime Environment (JRE)

<mark>JVM ถูกสร้างขึ้นเพื่อแปรง java byte code เป็น machine code</mark>

โดยแต่ละระบบปฏิบัติการจะมี JVM ที่ถูกออกแบบมาสำหรับระบบนั้น ๆ ซึ่งไม่ว่า byte code จะถูกคอมไพมาจากระบบปฏิบัติการอะไร ถ้าหาก JVM สมารถูกติดตั้งได้ ก็จะสามารถทำงานบนระบบนั้นได้

JVM ยังช่วยให้โปรแกรมมีความปลอดภัยมากขึ้นด้วย โดยในการทำงาน มันจะสร้างสิ่งที่เรียกว่า sandbox ซึ่งป้องกันไม่ให้โปรแกรมเรียกใช้งานทรัพยากรณ์แบบไม่จำกัด

โดยทั่วไปแล้ว การใช้งาน native code จะช้ากว่าการใช้งานแบบ machine code เนื่องจาก จำเป็นต้องมีตัวแปรภาษามาขั้นระหว่างการทำงานก่อน

(code) ---> (jvm) ---> (machine code)

(code) ---> (machine code)

*byte code เป็น native code*

แต่ในการทดสอบของ JVM พบว่าไม่ได้มี่ความแตกต่างกันมากในเรื่องของความเร็ว เนื่องจาก byte code ถูกออกแบบและทดสอบมาเป็นอย่างดี

ในช่วงแรกที่ภาษา java ถูกปล่อยออกมาให้ใช้งาน เราพบว่ามันมีความช้ากว่า C/C++ เป็นอย่างมาก เนื่องจากการทำงานแบบแปล byte code เป็น machine code มันจะแปลทีละคำสั่งทุกครั้งที่จะทำงาน ทำให้เกิดเทคโนโลยีใหม่ขึ้นมาที่มีชื่อว่า HotSpot

HotSpot จะสังเกตพฤติกรรมการรันโปรแกรม ดูว่า method ไหน / loop ไหน ถูกเรียกใช้งานบ่อย มันจะใช้สิ่งที่เรียกว่า Just-In-Time Compiler (JIT) ช่วยแปลชุดคำสั่งนั้นเป็น machine code เก็บไว้ ทำให้การทำงานทุกครั้งไม่จำเป็นต้องแปลชุดคำสั่งพวกนี้อีก สามารถส่งคำสั่งไปให้ cpu ไปทำงานได้ทันที

แต่เราจะไม่แปลงชุดคำสั่งทุกชุดไปเป็น machine code เนื่องจาก เราต้องการให้ชุดคำสั่งสามารถรันได้บนทุกอุปกรณ์ การจะแปลชุดคสั่งจาก byte code ไปเป็น machine code จะใช้เวลาคอมไพล์ + หน่วยความจำ ทำให้อาจะเกิดปัญหาในอุปกรณ์ขนาดเล็ก เช่น อุปกรณ์ในตู้ไมโครเวฟ ที่มีแรมและซีพียูที่มีพลังประมวลผลน้อย

## ก้าวไปไกลกว่าแอพเพล็ต (Moving Beyond Applets)
ตั้งแต่ java development kit 9 เป็นต้นไป java applets ถูกถอดออกจากระบบของ java เนื่องจากจำนวนบราวน์เซอร์ที่สนับสนุนนั้นลดน้อยลง

<mark>Deprecated หมายถึง ฟังก์ชันนั้นยังสามารถใช้งานได้ แต่ว่าล้าสมัยหรือถูกยกเลิกไปแล้ว</mark>

## การอัพเดทเวอร์ชันของ (Java A Faster Release Schedule)
Java เคยออกเวอร์ชันใหม่ทุก ๆ 2 ปี แต่ในปัจจุบันได้เหลือเพียง 6 เดือน เพื่อตอบสนองความต้องการในการเขียนโปรแกรมที่เปลี่ยนไป โดยแต่ละครั้งจะมีฟีเจอร์ที่แตกต่างกันออกเป็น 2 ประเภท คือ

1. Preview Features เป็น API หรือไวยากรณ์ใหม่ของภาษาที่ได้รับการออกแบบใช้งานอย่างสมบูรณ์แล้ว แต่อาจต้องปรับปรุงเพิ่มเติมก่อนจะนำไปใช้งานจริง
2. Incubator Modules เป็นฟิเจอร์ที่อยู่ในขั้นตอนการพัฒนา อาจมีฟีเจอร์นี้ในอนาคตหรือโดนลบออกก็ได้

และจนกว่าเวอร์ชันถัดไปจะออก เวอร์ชันที่พึ่งออกล่าสุดจะได้รับการสนับสนุนและปรับปรุงอยู่ โดยเราจะเรียกเวอร์ชันเหล่านี้ว่า Long Term Support (LTS)
